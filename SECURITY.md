# Security Policy

## Reporting a vulnerability

Please do not disclose a suspected vulnerability in a public Issue.

While this repository is private, report security issues to the repository owners through GitHub. Before the repository is made public, configure GitHub Private Vulnerability Reporting or add a dedicated security contact here.

Include a concise description, affected version, reproduction steps, potential impact, and any suggested mitigation. Do not attach personal photos, unredacted EXIF/GPS data, signing certificates, provisioning profiles, API tokens, PINs, or private keys.

## Supported versions

Security fixes are applied to the latest branch until a formal release support policy is published.
